# Live interactive prosthesis sim (MuJoCo WASM)

Real-time, in-browser MuJoCo sim of the generated prosthesis arm. Vendored from
[zalo/mujoco_wasm](https://github.com/zalo/mujoco_wasm); the arm scene is generated
from our `build_mjcf`.

## Run

```bash
# 1) (once) install deps — pulls the mujoco-js wasm + three.js
cd webdemo && npm install

# 2) (re)generate the arm scene from the current design
python3 ../scripts/demo/export_web_arm.py        # -> assets/scenes/arm.xml

# 3) serve it (any static server works; node modules are loaded relative)
python3 -m http.server 8011

# 4) open in a real browser (needs WebGL — works in Chrome/Safari, not headless)
open http://localhost:8011
```

## What you can do live

- **Actuator sliders** (Simulation ▸ Actuators) drive each joint in real time.
- **Drag any link** to poke the arm — applies a real force (`mj_applyFT`).
- **Pause / Reset / Reload** (Space / Backspace / Ctrl-L).
- Scene dropdown ▸ **Prosthesis Arm** (default) vs the stock example scenes.

## How it's wired

- `assets/scenes/arm.xml` — generated by `scripts/demo/export_web_arm.py`
  (`build_mjcf(DesignParams())` + a green target marker).
- `src/main.js` — `initialScene = "arm.xml"`.
- `src/mujocoUtils.js` — `arm.xml` added to the loader file list + scene dropdown.

To skin the moving arm with real CAD geometry later, export per-link STLs via
`CadBridge.export_arm(design)` and pass `mesh_dir=` to `build_mjcf` (the builder
already supports per-link `<mesh>` assets), then re-run the exporter.
